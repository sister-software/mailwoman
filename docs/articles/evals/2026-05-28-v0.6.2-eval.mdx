# v0.6.2 — Stage 3 corpus rebalance eval

The v0.6.2 release fixes the
[v0.6.1 dependent_locality regression](v0.6.1-error-analysis.mdx) via the corpus rebalance
recipe documented in
[street-supplement-architecture.md](../concepts/street-supplement-architecture.mdx). This
doc captures the empirical results.

This page is a **draft template** — the TBD fields land when the v0.6.2 100K eval
pipeline (`scripts/eval-v062-checkpoint.sh`) completes.

## Setup

- **Model:** v0.6.2 (`model-v062-step-100000-int8.onnx`)
- **Training corpus:** v0.4.0 base + synth-no-street shard (122K rows; 112K US + 10K
  DE/GB; 35% venue-adversarial)
- **Source-weight delta from v0.6.1:** `synth-street` 2.0 → 0.5; `synth-no-street` 1.0 (new)
- **Tokenizer:** v0.6.0-a0
- **Inference:** admin FST `fst-en-us.bin` + morphology FST (libpostal street_types,
  length-3-filtered, 1,707 canonicals)
- **Golden set:** `data/eval/golden/v0.1.2/` (4,561 entries)
- **v0-vs-neural harness:** `mailwoman/test/*.test.ts` (376 v0 assertions) +
  `data/eval/falsehoods/` (39 falsehoods rows) = 415 total
- **Gate baseline:** v0.6.0 default per-tag JSON
  (`/tmp/eval-v060-true-baseline.json`)

## Notes from the training run

(Filled in incrementally as the run progressed.)

- Throughput: **~16 steps/s on A100 80GB PCIe** (vs ~6 steps/s for the diagnostic on
  A100-SXM4-40GB). 100K steps ≈ 1h45min wall-clock.
- Loss trajectory: warmup → step 1000 train_loss=0.286 → step 2000 train_loss=0.097
  (first eval: val_loss=0.333, macro_f1=0.371) → step 8000 macro_f1=0.410, locality=0.862,
  region=0.725, street=0.984, house_number=0.999, postcode=1.000 → step 12000
  macro_f1=0.413.
- No NaN. CE-only training is stable (as expected — the bf16-NaN issue from the
  postmortem only affected CRF, which v0.6.2 leaves disabled per the NaN protocol).
- Early-eval gate at step 20K: **TBD** (dep_loc hallucination ceiling 100; abort + bump
  synth-no-street size if exceeded).
