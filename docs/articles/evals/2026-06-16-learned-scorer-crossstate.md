# Learned-scorer CROSS-STATE generalization (#603 Tier 2) — train TX, evaluate CA

_Generated by `scripts/record-matcher/learned-scorer-crossstate-eval.ts`. The GBT + LR are trained on 2000 TX NPIs (5641 records) and used to cluster 2000 held-out CA NPIs (5574 records) — a state the model never saw — through the same `resolveEntities` pipeline (FS baseline / GBT scorer / LR scorer), best F1 over a fine per-scorer threshold sweep. This is the generalization axis the within-state held-out-NPI A/B couldn't cover._

## Result — CA clustering F1 (GBT/LR trained on TX)

| scorer                     | precision | recall |        F1 |   ΔF1 vs FS | over-merged |
| -------------------------- | --------: | -----: | --------: | ----------: | ----------: |
| FS baseline (EM-fit)       |     10.4% |  26.4% |     15.0% |           — |         239 |
| logistic regression        |      7.7% |  71.1% |     13.9% |      -1.0pp |         279 |
| **gradient-boosted trees** |     58.8% |  25.4% | **35.5%** | **+20.5pp** |          47 |

**The GBT win GENERALIZES across states** — trained on TX, it still beats the FS baseline on CA clustering F1 (35.5% vs 15.0%, +20.5pp). The learned scorer isn't fitting TX-specific structure; the over-merge signal it learns transfers. This is the strongest evidence yet for the #603 production GBM — one model, trained once, helps a state it never saw.

## Caveats

A single train/eval state pair (TX→CA), one geocoded sample each, a compact pure-Node GBT (120 rounds, depth 3). The FS arm is the benchmark baseline (same model), so the comparison is fair. Absolute F1 differs from the within-state A/B because the eval population + over-merge density differ by state. NPI-as-truth is conservative. The within-state held-out-NPI A/B (`learned-scorer-clustering-eval.ts`) is the companion; together they bound the generalization question a production GBM must answer.\_
