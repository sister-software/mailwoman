# Switching from Pelias

[Pelias](https://pelias.io) is the modular, self-hosted geocoder — importers for OSM, OpenAddresses,
Who's On First, and more, feeding an Elasticsearch cluster. It's capable and it's a lot to operate: the
cluster, the importers, the gigabytes, the analysis chain you tune by hand.

Mailwoman keeps the parts you wanted from Pelias — self-hosted, open data, hierarchy-aware — and drops
the cluster. The hierarchy lives in SQLite (a Who's On First `ancestors` table), and a calibrated neural
model does the work the analysis chain did with rules.

## Mapping the move

Pelias has its own `/v1` response envelope, and Mailwoman does not ship a `/v1`-compatible package today.
The migration is by capability rather than by drop-in:

| Pelias             | Mailwoman                                     |
| ------------------ | --------------------------------------------- |
| `/v1/search`       | `@mailwoman/nominatim` `/search` (structured) |
| `/v1/reverse`      | `@mailwoman/nominatim` `/reverse`             |
| `/v1/autocomplete` | `@mailwoman/photon` `/api`                    |
| structured parsing | `@mailwoman/libpostal` `/parse`               |

```bash
npx @mailwoman/nominatim serve   # structured search + reverse, with annotations
npx @mailwoman/photon serve      # autocomplete (GeoJSON)
```

## What's different

- **No Elasticsearch, no import pipeline.** One SQLite gazetteer instead of a cluster you feed.
- **A model, not an analysis chain.** Parsing is a calibrated neural tagger, so you get [a confidence you
  can route on](./confidence-calibration.mdx) — and an OpenCage-style annotations block Pelias doesn't produce.

## What's missing

- **No `/v1` envelope yet.** If your client is hard-wired to Pelias's exact response, you'll adapt it to
  the Nominatim / Photon shapes (or a `/v1` adapter is a reasonable contribution).
- **Coverage and ranking** are their own long roads; Pelias's mature importer set still wins in places.

See [how Mailwoman compares](./how-mailwoman-compares) for the capability matrix.
