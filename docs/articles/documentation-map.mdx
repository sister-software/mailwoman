# Documentation map

The switcher at the top of every docs page has nine sections. Here's what each one is for.

- **Start here** — the front door: [install, your first parse](./getting-started.mdx), [current status](./status.mdx), [releases](./releases.mdx), and [how to contribute](./contributing.md).
- **Use Mailwoman** — [task-oriented recipes](./recipes/index.md): [batch geocoding](./recipes/batch-geocoding.md), [CSV ingest](./recipes/parallel-csv-ingest.md), [timezone lookups](./recipes/timezones.md), and the rest.
- **Understand** — [the why and how](./understanding/README.mdx): the problem Mailwoman solves, why address parsing is hard, [the alternatives](./understanding/alternatives/the-case-for-simple-geocoders.mdx), our approach, and [concept deep dives](./concepts/README.mdx).
- **Reference** — the contracts a build depends on: [Current scope & roadmap](./plan/SCOPE.mdx), [the `ComponentTag` schema](./plan/reference/SCHEMA.mdx), [the runtime pipeline stages](./plan/reference/STAGES.mdx), and the other lookup docs kept in sync with the code. The roadmap question routes here — the Archive's plan directory is history, not the plan of record.
- **Contribute** — runbooks for [working on the model](./plan/CONTRIBUTING_MODEL_WORK.mdx), the training data, and [operations](./plan/reference/OPERATIONS.mdx), if you're extending Mailwoman itself.
- **Archive** — closed engineering records: dated plans, phase-by-phase build history, and superseded reference docs, kept for provenance.
- **Eval reports** — per-model score reports and parity scorecards.
- **Retrospectives** — night-shift and project retrospectives.
- **Legal** — [licensing](./licensing/index.md), [data provenance](./licensing/data-provenance.md), and [privacy](./licensing/privacy.md).

## What to trust

Only **Reference** is kept current with the code. **Start here**, **Use Mailwoman**, and **Understand** teach you how to use and reason about Mailwoman, and if something there conflicts with Reference, Reference wins.

**Archive** holds closed, dated records, left as they were written rather than rewritten. An archived page keeps its original URL, so old links and bookmarks still resolve; you land on a page that says it's superseded, not a dead end.

**Eval reports** and **Retrospectives** are records too, maintained separately from this reorganization.

If you land on an Archive page from a search result or an old link, that's expected. Check **Reference** or **Start here** for the current version of whatever it describes.
