# Corpus v0.5.0 rebuild plan (2026-06-11 — scope frozen by the #519 review)

The four passengers, ruled and locked: **char-offset labels** (#519, from-source, parallel
arrays), **DE holdout** (already merged into `defaultHoldouts()` — activates automatically),
**ZCTA centroid fill** (#525), **FTS alias-bag boundary separator** (#523). Everything else is
v0.5.1. This doc is the integration order + the estimate scaffold; it becomes the build runbook
when scheduled.

## Order of operations

1. **Format change, code-first** (corpus/ + corpus-python, no build yet):
   - `alignRow` emits `span_starts[]/span_ends[]/span_tags[]` (deleting the token quantization —
     the char offsets already exist internally). Build-time NFC assertion per row (the ruling
     replaced the converter's Unicode class with this).
   - `encode_row` consumes spans directly (promoting the internal per-char array to the input
     contract); the glue augmentation and #511 relabel pass re-target spans (both simplify — the
     relabel becomes char arithmetic).
   - Per-piece channel invariance gates: anchor + gazetteer painting asserted identical on a
     fixture set of punctuation-free rows (the consult keeper, now an encode-time test).
   - Every shard builder migrates (affix, unit, intersection, po_box/cedex+au/nz, country,
     german). Builder audits move to RAW-surface comparison (the dotted-blind-spot lesson).
2. **ZCTA join** (#525): Census ZCTA gazetteer file → `build-pilot-anchor-lookup.py` + the WOF
   postcode row builders, `source` provenance per filled row. Acceptance: anchor placeholder
   rate 38% → single digits; "90210" circles in the demo.
3. **FTS separator** (#523): boundary-preserving separator in `buildPlaceSearchFts`'s
   `GROUP_CONCAT` + the matching exact-tier check; slim-DB rebuild rides the same artifact pass.
4. **Pilot build** (1 shard, ~1M rows): calibrates the full-build wall-clock estimate AND runs
   the invariance gates against real data before committing days of compute. THE GO/NO-GO GATE.
5. **Full from-source build** (~677M rows + the v0.4.x overlay shards re-emitted natively).
   DE holdout materializes here (SPLIT_MANIFEST picks up Saarland + Mecklenburg-Vorpommern).
6. **Validation campaign**: invariance on sampled converted-equivalent rows, lint-corpus-shard,
   a DE honest-eval slice trust check (≥1000 rows), then the first new-format training run with
   the PRE-REGISTERED bridge-retirement gate (dotted po_box ≥ 89.1 bridge-OFF, FR postcode
   ≥ 99.5, affix floors unchanged, over-merge precision floor, #518 as the punctuation lens).
