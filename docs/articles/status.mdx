# Status

This page is the single source of truth for what works in Mailwoman right now.

:::info[Verified as of 2026-07-14 тАФ release 6.1.0]
The current npm release is **6.1.0** (`npm view mailwoman version`; `git tag` confirms `v6.1.0`), a
minor that promotes the trained model to **the v261 span-boundary line**. The prior release, 6.0.0,
was a breaking, code-only major тАФ the Hono API-surface cutover. All four HTTP surfaces (the native
`mailwoman serve` plus the photon/nominatim/libpostal drop-ins) run on one shared Hono stack, and
the express server is deleted in full with no compatibility shim. See
[Releases & capabilities](./releases.mdx) for both rows.

**The current model is v261** (`v2.6.1-span-boundary-full`,
[#727](https://github.com/sister-software/mailwoman/issues/727) stage-1) тАФ an init_from fine-tune
of the stable `v257` fragment-campaign line plus a training-only span-boundary auxiliary head
(discarded at export; the shipped ONNX is the plain BIO tagger whose encoder absorbed the boundary
pressure). It shrinks the regionтЖТstreet boundary-absorption class (US regionтЖТstreet flips 5тЖТ2 vs
v257), parity street 0.5281 (+13pp vs v241), gauntlet regression+metamorphic PASS (Dublin pin
held). Golden country recall 82.0% vs shipped 88.6% (тИТ6.6pp) ships as a **documented cosmetic gate
exception** ([#1104](https://github.com/sister-software/mailwoman/issues/1104)): the drop
concentrates entirely on 220 of 224 non-postal WOF-admin-hierarchy golden rows; on real-postal
addresses country recall and precision are identical across the whole fragment lineage (v241,
v257, v261 тАФ a DeepSeek-signed falsifier). The tokenizer is unchanged, `v0.9.0-multisplice`
(73,143 pieces). See the [v261 promotion + gate exception](./evals/2026-07-14-v261-promote-country-gate-exception.md)
note for the full gate table.

**The per-tag tables below are frozen at the 2026-07-02 full re-score of the earlier 5.0.0/v4.15.0
line** (the #885 measurement re-anchor). Six model releases have shipped since without a fresh
full re-score (5.1.0 тЖТ 5.4.0, then 5.9.0, then 6.1.0); their measured deltas are narrated
release-by-release тАФ not folded into a new scorecard тАФ on
[Releases & capabilities](./releases.mdx), which is the current per-release source of truth. Treat
the [parity scorecard](./evals/competitive-parity/parity-scorecard-2026-07-02.md) below as a
historical baseline, not the shipped model's live per-tag truth.
:::

## Confidence

Every node carries a confidence score; pass a calibrator (isotonic, shipped per-locale in the
weights bundle as `calibration.json`) to make those scores honest probabilities тАФ see
[Confidence calibration](./concepts/confidence-calibration.mdx) for what the numbers mean.

## Browser demo

The live demo at [mailwoman.sister.software/demo](https://mailwoman.sister.software/demo) runs the
int8 model (37.6 MB, the v261 line) via `@mailwoman/neural-web` plus a **byte-range global
gazetteer** тАФ the FTS-free "candidate" table (~530 MB, queried over `sql.js-httpvfs`; a resolve
touches ~12 range fetches regardless of file size) with global admin coverage + **US + EU postcode
resolution** and a debounced FST place-autocomplete in the search box. US **street-level
(rooftop)** resolves nationwide тАФ all 50 states + DC + VI тАФ via per-state situs/interp shards
(124.9M address points, [#735](https://github.com/sister-software/mailwoman/issues/735)); FR
rooftops resolve via the hosted BAN situs shard with a street-tier fallback (shipped 5.10.0, see
[Releases & capabilities](./releases.mdx)). Artifacts are served byte-range from R2. The demo
calls the classifier's argmax path directly; joint reconcile is not active in the demo.

## Model line (July 2026)

The shipped weights are the **v261 line** (`v2.6.1-span-boundary-full`, promoted 6.1.0 тАФ
[#727](https://github.com/sister-software/mailwoman/issues/727) stage-1). Six model releases
shipped between the 5.0.0/v4.15.0 line (documented in past versions of this page) and today, each
held to the **shipped assembled coordinate or a targeted repro**, not raw label-F1: v5.1.0
(`v0.6.0-bsplice` tokenizer splice) fixed Slavic diacritic fragmentation тАФ CZ/PL/SK/SI wrong-city
roughly halved, US byte-identical; v5.2.0 (`v0.7.1-nsplice`) extended the splice to Nordic
diacritics (SE resolve 83.8 тЖТ 90.2%); v5.3.0 (`v2.2.0-full-family`, from scratch) fixed the FR
street fragmentation the v5.2.0 splice had shipped ungated (bare 16/40 тЖТ 34/40, anchored 13/40 тЖТ
37/40); v5.4.0 (`v2.3.0-nl-postcode`, fine-tune) fixed a Dutch digits-first postcode misread that
was pulling `1012 LG Amsterdam` into the US situs tier; v5.9.0 (`v2.4.1-fr-nsplice-ft`, fine-tune)
gave the merged FR diacritic pieces gradient after a training-free vocab splice was falsified for
regressing FR streets, cutting the accent mangle rate 23.5% тЖТ 17.7%; v6.1.0/v261
(`v2.6.1-span-boundary-full`, fine-tune off the stable v257 fragment-campaign line) trained a
span-boundary auxiliary head to shrink the regionтЖТstreet boundary-absorption class (US regionтЖТstreet
flips 5тЖТ2 vs v257, parity street +13pp vs v241), shipping through a documented cosmetic gate
exception on golden country recall ([#1104](https://github.com/sister-software/mailwoman/issues/1104) тАФ
real-postal country recall and precision are unaffected). Full per-release evidence lives on
[Releases & capabilities](./releases.mdx), which this page points to for anything shipped after
the last full re-score. The ordered work queue to the full geocoder is
[epic #488](https://github.com/sister-software/mailwoman/issues/488).

## What is not supported today

- Multi-locale ensemble (en-US + fr-FR weights in the same runtime).
- Korean addresses (no open data path). Japanese addresses **are** supported via the
  postcode-route resolver (Geographic Rule Engine).

Three items that used to live on this list have shipped and are covered elsewhere on this page:
nationwide US rooftop geocoding (all 50 states + DC + VI, see **Browser demo** above,
[#735](https://github.com/sister-software/mailwoman/issues/735)), reverse geocoding (`mailwoman
reverse`, see the **Runtime pipeline** table above, [#484](https://github.com/sister-software/mailwoman/issues/484)),
and the production-hardened server API (batch, observability, `RemoteResolver`, versioned data
switchover тАФ shipped v4.7.0, [#485](https://github.com/sister-software/mailwoman/issues/485); the
6.0.0 Hono cutover retired the last express-era code path, see
[Releases & capabilities](./releases.mdx)).
