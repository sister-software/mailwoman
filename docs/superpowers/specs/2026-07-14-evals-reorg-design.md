## Problem

## Constraints (from the codebase)

- The evals sidebar is **autogenerated** (`docs/sidebars.ts`, `dirName: "evals"`) â€” subdirectories
  become nav categories automatically; category order comes from `_category_.json` `position`.
- **`onBrokenLinks: "throw"`** (`docusaurus.config.ts`) â€” the docs build fails on any dangling
  internal link. This is the completeness gate: a green `yarn build` proves every link resolves.
- Moving a file changes its Docusaurus doc id â†’ URL. Docusaurus resolves relative markdown links by
  **source file path**, so every relative cross-link to a moved file must be rewritten regardless of
  URL policy.

## Decisions (operator-approved)

1. **Taxonomy:** topic-based, one level deep. Sidebar **ordered as a capability narrative** â€” what
   the system does / how it improved up top; one-off experiments and operational logs sink lower.
2. **Postmortems:** night-shift session logs get their **own** category; a **retrospectives**
   category rides at the top. Both stay inside the `evals` sidebar (the top-level `retrospectives`
   section is out of scope for an evals-only reorg).
3. **URLs:** follow the new structure (`/docs/evals/<category>/<name>`). Internal docs stay
   link-consistent (build-enforced); external/off-site breakage is acceptable. No `slug` pinning.

## Implementation

1. Create the 9 subdirs + `_category_.json` (label + position) each. `git mv` per `move-map.tsv`;
   move the 2 loose data files into `data/`; fold the stray `v0.5.4/` into `model-versions/`.
   Add a root `evals/index.mdx` (`sidebar_position: 0`) that links each category, plus an
   `index.mdx` landing in **each** category dir. Docusaurus auto-detects a folder's `index.mdx`
   as the category's link page, which creates the `/docs/evals/<cat>/` route the root index links to.
   (`_category_.json` `link: {type: generated-index}` was tried first and produced no routes in this
   3.10 + autogenerated-sidebar setup â€” a real index doc is the reliable mechanism.)

   - inbound `../evals/x` from `concepts/`, `research/`, `retrospectives/`, `plan/` â†’ `../evals/<cat>/x`;
   - absolute `/docs/evals/x` â†’ `/docs/evals/<cat>/x`.
3. `docusaurus.config.ts`: repoint navbar "Eval reports" link; widen exclude globs
